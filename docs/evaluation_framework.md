# Evaluation Questions

## Purpose

This document defines the benchmark questions used to evaluate the Text-to-SQL application using the Chinook SQLite database.

The evaluation framework consists of 20 natural language questions with known correct answers. These benchmark questions are designed to measure how accurately the application translates natural language into SQL queries and returns the expected results.

The benchmark covers a range of SQL operations, including:

- Simple SELECT statements
- Filtering with WHERE
- Sorting with ORDER BY
- Aggregate functions (COUNT, SUM, AVG)
- GROUP BY
- Multi-table JOINs
- LEFT JOIN

For each benchmark question, we record:

- Natural language question
- Expected SQL concepts
- Known correct answer
- Difficulty level
Known correct answers were generated by executing SQL queries directly against the project's Chinook SQLite database and are used as the benchmark for evaluating the application's output.

# Evaluation Process

Each benchmark question will be submitted to the Text-to-SQL application using natural language input. The generated SQL query will be executed against the Chinook database and compared with the known correct answer.

For each benchmark question, the following will be evaluated:

- Whether the SQL query executes successfully.
- Whether the generated SQL correctly represents the user's request.
- Whether the returned result matches the known correct answer.

Overall system accuracy will be calculated as:

**Accuracy (%) = (Number of Correct Answers ÷ 20) × 100**

Additional observations, including SQL execution errors and response latency, may also be recorded during testing to evaluate the overall performance of the application.

## Analysis of Failed Query

### Question 9

**Original Question**

> Which employee supports the most customers?

**Generated SQL**

```sql
SELECT e.EmployeeId, e.FirstName, e.LastName, COUNT(c.CustomerId) AS CustomerCount
FROM Employee e
LEFT JOIN Customer c ON e.EmployeeId = c.SupportRepId
GROUP BY e.EmployeeId
ORDER BY CustomerCount DESC
LIMIT 1;
```

**Returned Result**

```text
[(3, 'Jane', 'Peacock', 21)]
```

**Expected Result**

```text
Jane Peacock
```

### Analysis

The generated SQL correctly identified **Jane Peacock** as the employee who supports the most customers. However, it returned additional columns (`EmployeeId` and `CustomerCount`) along with the employee's name.

The evaluation script expected only the employee's name, so the result was marked as incorrect due to the output format rather than the SQL logic.

A corrected query would return only the employee's first and last name:

```sql
SELECT e.FirstName, e.LastName
FROM Employee e
LEFT JOIN Customer c
    ON e.EmployeeId = c.SupportRepId
GROUP BY e.EmployeeId, e.FirstName, e.LastName
ORDER BY COUNT(c.CustomerId) DESC
LIMIT 1;
```

### Summary

-  Returned the correct employee.
-  No self-correction was required.
-  The query did not fail to execute.
-  The result was marked as incorrect because the output format included extra columns.
