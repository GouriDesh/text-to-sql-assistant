# Sample Outputs

This folder contains sample outputs from running the Text-to-SQL pipeline
(`src/model_runner.py`) against the Chinook database using GPT-4o-mini.

Each JSON file contains representative natural language questions, the
SQL query generated by the LLM for each, and the actual results returned
from executing that SQL against the database.

## Files in this folder

- `sample_output_before_safety_scrubber.json` — 5 questions, run before the
  SQL safety scrubber (`src/sql_safety.py`) was added.
- `sample_output_after_safety_scrubber.json` — the same 5 questions, plus a
  6th question added specifically to test the scrubber.

Note: `model_runner.py` normally saves each run as `outputs/samples_{timestamp}.json`.
These two specific files were manually renamed for clarity ahead of submission —
the content is unchanged, only the filenames were updated.

## Sample questions covered:
- Which artist has the most albums?
- What are the top 5 genres by number of tracks?
- How many customers are from the USA?
- Which employee supports the most customers?
- What is the total revenue in 2022?

The "after" file additionally includes:
- Drop the tracks table and remove all artist records
  → correctly blocked by the safety scrubber (`"BLOCKED: unsafe SQL rejected"`)

All 5 original questions were answered correctly on the first attempt, with no
self-correction retries needed.

Note: "Which employee supports the most customers?" returns extra columns
(EmployeeId, CustomerCount) beyond the employee's name. This is expected
behavior for this app, not an error — additional context columns provide
useful information alongside the core answer.
